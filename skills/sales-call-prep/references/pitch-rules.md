# Pitch Rules

## Service-First Rule

If recent history contains unresolved issues, product rejection, email bounce, category mismatch, Buy Lead consumption concerns, export lead visibility concerns, or servicing follow-up, start with issue resolution before any upsell.

## Category/Product Rule

Use enriched category data to make the pitch specific:

- Use the top MCAT when rank, demand, and product count are strong.
- Mention primary products only when they are present in the seller record.
- Use support categories for expansion only after the main category issue is addressed.
- Do not pitch a category that conflicts with the seller's preference or unresolved complaint.

## Evidence Rule

Every pitch must include at least one seller-specific reason:

- recent call or meeting note
- repeated pain point
- performance metric
- package or service indicator
- MCAT/product demand signal
- catalog or product-quality issue

## Objection Rule

Handle objections by acknowledging the issue first, then giving a concrete next step. Avoid hard-selling when the seller's previous complaint is unresolved.

## Safety Rule

Do not reveal unmasked contact details. Use masked values exactly as provided. Do not invent payment, ledger, receipt, WebERP, or package-renewal details.

## Hinglish Example

Sir, pehle main aapka export Buy Lead visibility issue verify karwa deta hoon, kyunki recent history me ye concern repeat hua hai. Uske baad Steel Drag Chain category aur primary products ko optimize karke relevant enquiries aur package value par discussion karenge.
