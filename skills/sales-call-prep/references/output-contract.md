# Output Contract

Use these sections exactly when producing each output type.

## Pre-Call Brief

```markdown
## Client Snapshot
## Account And Performance
## Pain Points
## Category / Product Signals
## Recommended Call Objective
## Suggested Opening
## Risks / Avoid Points
## Evidence
## Sources
```

## Sales Pitch

```markdown
## Opening
## Context Acknowledgement
## Issue Resolution Plan
## Category / Product Hook
## Value Pitch
## Likely Objections
## Next Step
## Follow-Up Note
## Evidence
## Sources
```

## Chat Answer

```markdown
## Direct Answer
## Suggested Salesperson Action
## Evidence
## Sources
```

## Style Requirements

- Keep answers short enough for a salesperson to read before or during a call.
- Prefer bullets for actions and evidence.
- Put the most important action before explanation.
- Use concrete seller facts instead of generic sales language.
- Keep Evidence separate from Sources.
- Do not show raw JSON to the salesperson.
